# TaskMaster v3.0.0 升级说明 (Release Notes)

尊敬的用户/审核人员，您好！

TaskMaster 从 v2.9.0 跃升至 v3.0.0，这是我们发布以来最大的一次底层重构与视觉升级。本次更新重点解决了旧版本在数据存储容量、事件内存泄漏上的痛点，并引入了全新的极简设计语言与自动备份机制。

## 🚀 核心升级亮点

### 1. 突破存储限制：双驱混合存储架构
- **背景**：旧版 (v2.9.0) 将所有任务数据强制存储在 `chrome.storage.sync` 中。由于浏览器的同步配额极小（单项 8KB，总容量 100KB），导致重度用户极易触发 `QUOTA_BYTES_PER_ITEM` 错误，造成数据丢失。
- **升级方案**：v3.0.0 引入了**“核心数据 Sync 同步 + 全量数据 Local 本地存储”**的双驱架构。
- **平滑迁移**：当扩展自动升级到 v3.0.0 后，后台脚本会自动识别旧版的 Sync 数据，并将其完整、无损地迁移至大容量的 Local 存储中，释放 Sync 空间，随后只将基础字段提取至 Sync 用于跨端合并。

### 2. 首创机制：Service Worker 自动静默备份
- **功能说明**：用户常常忘记手动备份数据。现在，我们利用 Manifest V3 的 `chrome.alarms` API，在后台实现了一套 24 小时定时滚动备份机制。
- **快照管理**：每天自动生成一份全量数据的 JSON 快照存入 Local 中（默认保留最近 7 天的 7 份快照）。
- **热更新引导**：本次从 2.9.0 升级后，扩展会在后台**强制执行一次全量备份**，并自动打开设置页，通过高亮弹窗强烈引导用户手动导出一份本地 JSON 文件，确保历史数据的绝对安全。

### 3. 视觉与交互重塑 (Typography First)
- **极简美学**：彻底抛弃了旧版本中泛滥的卡片设计、阴影和生硬的圆角高亮边框。新版全面拥抱由留白、字重驱动的高级品牌风。
- **动态交互**：重写了底层的 DOM 更新逻辑，消除了全量 `innerHTML` 重绘导致的滚动条丢失和折叠状态重置问题；所有悬浮（Hover）和聚焦（Focus）动作都加入了基于 `cubic-bezier(0.16, 1, 0.3, 1)` 的苹果级阻尼缓动。
- **多语言重构**：全面接入 `_locales` (i18n) 国际化标准目录，原生支持英文 (en) 和简体中文 (zh_CN) 动态切换。

### 4. 性能优化与安全修复
- **修复事件监听泄漏**：将原先在每次渲染流中重复绑定的 DOM 事件（导致数百次重复触发点击），剥离至初始化阶段统一进行事件委托，大幅降低了 CPU 开销。
- **防范 XSS 攻击**：增强了底层的 HTML 字符串拼接转义逻辑，不仅对文本节点，同时也对标签的属性注入点（如单双引号）进行了全面过滤。
- **修复加解密崩溃**：修复了旧版本 Options 页面在导出加密数据时，错误调用同步加密函数导致的 `ReferenceError` 崩溃。

---

**Edge 团队审核备注：**
1. 本扩展完全遵循 Manifest V3 规范。
2. 扩展新申请了 `"downloads"` 权限，仅用于支持用户手动导出其任务数据（生成 JSON 备份文件）以及自动备份的可视化恢复。
3. 扩展所有请求与存储均在用户本地及 Edge Sync API 中完成，无任何第三方追踪代码。
